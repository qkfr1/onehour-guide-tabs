# guide writing principles

원아워 가이드 문서 작성 시 적용하는 원칙입니다.

***

## 1. 톤앤매너

| 상황    | 표현                  |
| ----- | ------------------- |
| 설명    | \~합니다               |
| 행동 유도 | \~해주세요              |
| 주의 환기 | \~됩니다 (수동형으로 결과 강조) |

***

## 2. 문서 구성

### 필수 항목

| 순서 | 항목          | 설명               |
| -- | ----------- | ---------------- |
| 1  | 페이지 제목 (H1) | 기능명 또는 작업명       |
| 2  | 기능 설명       | 1\~2문장으로 "이게 뭔지" |
| 3  | 이럴 때 사용하세요  | 실제 상황 예시 2\~3개   |
| 4  | 접속 방법       | 해당 화면까지 가는 경로    |
| 5  | 단계별 방법      | 번호 순서로 행동 안내     |

### 선택 항목

| 항목       | 포함 기준             |
| -------- | ----------------- |
| 화면 구성    | 복잡한 화면일 때 (표로 정리) |
| 고급/AI 기능 | 편의 기능이 있을 때       |
| 주의사항/팁   | 실수하기 쉬운 부분 있을 때   |
| 자주 묻는 질문 | 채널톡 문의 2회 이상      |
| 관련 문서 링크 | 연관 기능 있을 때        |
| 이미지/GIF  | 아래 기준 참고          |

***

## 3. 도입부 원칙

### ❌ 바로 사용법 시작 금지

```
1. 수업관리에서 수정할 수업을 선택합니다.
```

### ✅ 기능 설명 + 상황 예시 먼저

```
# 문장 수정 방법

수업에 포함된 영어 문장과 번역을 편집하는 기능입니다.

### 이럴 때 사용하세요
• 원마켓에서 받은 수업의 오타 수정
• 우리 학원 스타일로 번역 변경  
• 직접 만든 문장 추가
```

***

## 4. 용어 통일

### 금지 → 통일

| ❌ 금지 | ✅ 통일 |
| ---- | ---- |
| 통시통역 | 동시통역 |
| 쉬는 날 | 휴일   |

### 전문 용어 → 쉬운 말

| ❌ 전문 용어 | ✅ 쉬운 말       |
| ------- | ------------ |
| 포커스됩니다  | 커서가 깜빡입니다    |
| 드롭다운    | 펼침 메뉴 / ▼ 버튼 |
| 툴바      | 상단 버튼 모음     |
| 팝업/모달   | 확인 창 / 입력 창  |
| 토글      | 켜기/끄기 스위치    |

{% hint style="info" %}
**원칙**: 원장님이 전화로 물어봤을 때 설명할 수 있는 단어를 사용합니다.
{% endhint %}

## 5. UI 요소 표기

| 요소     | 표기법    | 예시                       |
| ------ | ------ | ------------------------ |
| 버튼/메뉴명 | **볼드** | **과제 생성** 버튼을 클릭해주세요     |
| 탭/섹션명  | **볼드** | 상단 **문장** 탭을 선택합니다       |
| 입력값/예시 | `코드블록` | 시작일에 `2024-01-01`을 입력합니다 |
| 경로     | > 구분   | 수업관리 > 수업 수정 > 문장 탭      |

***

## 6. 이미지 원칙

### 필수 상황

| 상황          | 예시                  |
| ----------- | ------------------- |
| 화면 전체 구성 설명 | 에디터 화면 레이아웃         |
| 버튼 위치 언급    | "상단 **전체 삭제** 버튼"   |
| 상태 변화 설명    | "저장 버튼이 주황색으로 변합니다" |
| 팝업/확인 창 설명  | 삭제 확인 창             |

### 이미지 규칙

* 강조 부분: 빨간 박스 또는 화살표
* 순서 표시: ①②③ 번호
* 캡션: 필요할 때만 (선택)

***

## 7. 주의사항 박스 활용

```
{% hint style="info" %}    → 알아두면 좋은 정보
{% hint style="warning" %} → 실수하기 쉬운 부분  
{% hint style="danger" %}  → 데이터 손실 등 치명적
```

| 유형      | 사용 상황   | 예시                            |
| ------- | ------- | ----------------------------- |
| info    | 조건부 기능  | "PDF/이미지 수업에서만 페이지 번호가 표시됩니다" |
| warning | 덮어쓰기/변경 | "이미 번역이 있는 문장도 덮어쓰기 됩니다"      |
| danger  | 삭제/복구불가 | "저장하지 않은 모든 수정 사항이 사라집니다"     |

***

## 8. FAQ 작성 기준

### 포함 기준

* 채널톡 문의 2회 이상
* "왜 안 되지?" 막히는 상황
* 기능 제한 사항

### 작성 형식

```
Q. [사용자가 겪는 증상 그대로]
A. [원인] + [해결법]
```

### 예시

```
Q. 저장 버튼이 비활성화되어 있어요.
A. 빈 문장, 빈 번역, 빈 페이지 번호가 있으면 저장되지 않습니다. 
   모든 칸을 채운 후 다시 시도해주세요.
```

## 9. 문서 품질 체크리스트

새 문서 작성 또는 기존 문서 수정 시 확인해주세요.

* [ ] "이 기능이 뭔지" 설명이 있는가?
* [ ] "이럴 때 사용하세요" 상황 예시가 있는가?
* [ ] 버튼 위치 언급할 때 스크린샷이 있는가?
* [ ] 전문 용어 대신 쉬운 말을 썼는가?
* [ ] 처음 보는 원장님이 혼자 따라할 수 있는가?
* [ ] 채널톡 자주 오는 질문이 FAQ에 있는가?
* [ ] AI/편의 기능이 잘 보이게 배치되었는가?
